# 🏫 ระบบเช็คชื่อและคัดกรองสุขภาพนักเรียนอัจฉริยะ (Smart Attendance System)

ระบบลงเวลาเรียนของนักเรียนผ่านกล้องเว็บแคมด้วย **ปัญญาประดิษฐ์ Face Recognition (สแกนใบหน้า) และระบบสแกน QR Code สำรอง** พร้อมด้วยระบบ **คัดกรองสุขภาพเบื้องต้น (วัดไข้/ไอ/ปกติ)** แจ้งเตือนผู้ปกครองแบบเรียลไทม์ผ่าน LINE Notify / LINE OA และระบบส่งรายงานสรุปสถิตินักเรียนของแต่ละห้องเรียนเข้ากลุ่มไลน์คุณครูและผู้ปกครองโดยอัตโนมัติประจำวัน

แอปพลิเคชันนี้ออกแบบมาให้ทำงานได้ **ฟรี 100%** ไม่มีค่าธรรมเนียมรายเดือน หรือค่าใช้จ่ายด้านเซิร์ฟเวอร์ และที่สำคัญคือ **คุณครูท่านอื่นสามารถคัดลอก (Make a Copy / Fork) นำไปตั้งค่าและใช้งานในโรงเรียนหรือห้องเรียนของตนเองได้ง่าย!**

---

## 🚀 ฟีเจอร์หลักของระบบ (Core Features)

1. **📸 ระบบเช็คชื่อแบบ Multi-Input (Face AI / QR / Barcode / เครื่องยิงฮาร์ดแวร์):** เช็คชื่อเข้าเรียนด้วยกล้องเว็บแคมผ่านระบบ AI จดจำใบหน้า, สแกน QR Code/Barcode ผ่านกล้อง หรือเชื่อมต่อเครื่องยิงบาร์โค้ด (USB/Bluetooth) ยิงรหัสได้จากพื้นหลังโดยไม่ต้องคลิกหน้าจอ พร้อมช่องป้อนรหัสแมนนวลกรณีเครื่องยิงขัดข้อง
2. **🩺 ระบบคัดกรองสุขภาพสั้นขณะลงเวลา (Health Screening Integration):** บันทึกอุณหภูมิร่างกายและสถานะคัดกรอง (ปกติ 🟢, มีอาการไอ 🟡, ตัวร้อน/มีไข้สูง 🔴) พร้อมตัวนับถอยหลัง 3 วินาทีสำหรับบันทึกอัตโนมัติเพื่อความรวดเร็ว ข้อมูลจะแนบไปใน LINE แจ้งเตือนทันที
3. **📊 ระบบสรุปยอดประจำวันอัตโนมัติเข้ากลุ่ม LINE (Daily Attendance Summary Bot):** ยิงข้อความรายงานยอดสถิติเข้าเรียนของวันนั้น ๆ เข้ากลุ่ม LINE ครูและผู้ปกครองโดยอัตโนมัติ ณ เวลาที่กำหนด (เช่น 08:30 น.)
4. **💳 ระบบออกบัตรนักเรียนดิจิทัล (Digital Student ID Badge):** ออกบัตรนักเรียนแบบกระจกฝ้า (Glassmorphism) มี QR Code และบาร์โค้ด (Code 128) ประจำตัวแสดงผลอยู่บนหน้าบัตร พร้อมปุ่มกดสั่งพิมพ์แยกเฉพาะตัวบัตรได้อย่างสวยงาม
5. **🚨 ระบบวิเคราะห์พฤติกรรมขาดเรียนเชิงลึก (AI Early Warning):** แจ้งเตือนเด็กกลุ่มเสี่ยงขาดเรียนบ่อย เสี่ยงหลุดออกจากระบบการศึกษา พร้อมคำแนะนำสำหรับครูประจำชั้น
6. **🔒 ระบบสิทธิ์แยกห้องเรียน (Classroom Access Control):** ล็อกสิทธิ์ครูประจำชั้นไม่ให้เห็นหรือแก้ไขห้องเรียนอื่น รวมถึงการล็อกหน้าความปลอดภัยสูงสุดสำหรับผู้ดูแลระบบ (Admin Guard)
7. **🔄 ระบบทำงานแบบออฟไลน์ (Offline Database & Auto-Sync):** หากเน็ตโรงเรียนหลุด ระบบจะยังสามารถสแกนจำหน้าเด็กในเครื่อง และจะบันทึกคิวไว้ในเบราว์เซอร์เพื่อซิงค์ส่งแจ้งเตือนย้อนหลังเข้า LINE ทันทีเมื่อต่อเน็ตติด
8. **🗄️ รองรับระบบฐานข้อมูล 2 รูปแบบ (Dual Database Engine):**
   * **Google Sheets (เริ่มต้นง่าย):** จัดเก็บข้อมูลลงตาราง Google Sheets ของคุณครูโดยตรง เหมาะสำหรับห้องเรียนเดี่ยว
   * **Supabase / PostgreSQL (ความเร็วสูง แนะนำสำหรับทั้งโรงเรียน):** จัดเก็บข้อมูลในฐานข้อมูลคลาวด์ความเร็วสูง รองรับข้อมูลปริมาณมหาศาล และมีระบบเก็บรูปถ่ายใบลาโดยเฉพาะ

## 🛠️ คู่มือการติดตั้งระบบฉบับจับมือทำสำหรับคุณครู (Step-by-Step Installation Guide)

คุณครูท่านใดที่สนใจนำระบบเช็คชื่อนี้ไปใช้เป็นส่วนตัวในห้องเรียนหรือโรงเรียนของตนเอง สามารถจัดเตรียมและติดตั้งระบบทั้งหมดได้ฟรี 100% ตามขั้นตอนจับมือทำดังนี้ครับ:

### ส่วนที่ 1: การทำสำเนา (Copy) และเตรียมระบบฐานข้อมูล

คุณครูสามารถเลือกใช้งานฐานข้อมูลได้ 2 รูปแบบตามความเหมาะสม:

#### ทางเลือก A: ระบบฐานข้อมูล Google Sheets (สะดวกและง่ายที่สุด)
1. **สร้างไฟล์แผ่นงาน Google Sheets ใหม่:** ให้เปิด Google Sheets ของบัญชี Google ของตนเองขึ้นมา 1 ไฟล์
2. **สร้างแผ่นงานย่อย (Sheet Tabs) ทั้งหมด 4 แผ่นงาน:** ตั้งชื่อแผ่นงานย่อยด้านล่างและกรอกชื่อหัวตารางในแถวที่ 1 (Row 1) ให้ตรงตามนี้:
   * **แผ่นงานที่ 1 ตั้งชื่อว่า `students`** 
     *(หัวตารางแถวแรกในคอลัมน์ A ถึง M: `id`, `name`, `email`, `faceDescriptor`, `consentGiven`, `registeredAt`, `classroom`, `level`, `parentLineId`, `avatarUrl`, `bloodGroup`, `emergencyPhone`, `medicalAlert`)*
   * **แผ่นงานที่ 2 ตั้งชื่อว่า `attendance`** 
     *(หัวตารางแถวแรกในคอลัมน์ A ถึง J: `id`, `studentId`, `studentName`, `studentEmail`, `timestamp`, `confidence`, `classroom`, `status`, `temperature`, `healthStatus`)*
   * **แผ่นงานที่ 3 ตั้งชื่อว่า `leaves`** 
     *(หัวตารางแถวแรกในคอลัมน์ A ถึง N: `id`, `studentId`, `studentName`, `classroom`, `startDate`, `endDate`, `type`, `reason`, `status`, `submittedAt`, `contactEmail`, `contactLine`, `contactPhone`, `evidenceUrl`)*
   * **แผ่นงานที่ 4 ตั้งชื่อว่า `settings`** 
     *(หัวตารางแถวแรกในคอลัมน์ A และ B: `key`, `value`)*
3. **ติดตั้งสคริปต์ระบบ (Google Apps Script):**
   * ในหน้า Google Sheets นั้น ให้กดที่เมนู **ส่วนขยาย (Extensions) > Apps Script**
   * ลบโค้ดเริ่มต้นออกให้หมด แล้วคัดลอกโค้ดทั้งหมดจากไฟล์ [`google_apps_script.js`](./google_apps_script.js) ในโครงการนี้ไปวางแทนที่ทั้งหมด จากนั้นกด **บันทึก (Save)**
4. **เปิดใช้งานสคริปต์ให้เป็นเว็บแอป (Deploy Web App):**
   * คลิกที่ปุ่ม **การทำให้ใช้งานได้ (Deploy) > การทำให้ใช้งานได้ใหม่ (New Deployment)**
   * เลือกประเภทเป็น **เว็บแอป (Web App)** (คลิกไอคอนฟันเฟืองด้านซ้ายเพื่อเลือก หากยังไม่มีประเภทขึ้นมา)
   * ตั้งค่าตัวเลือกที่สำคัญดังนี้:
     * *เรียกใช้งานในฐานะ (Execute as):* เลือก **ฉัน (บัญชีอีเมล Google ของคุณครูเอง)**
     * *ผู้มีสิทธิ์เข้าถึง (Who has access):* เลือก **ทุกคน (Anyone)** *(เพื่ออนุญาตให้แอปเช็คชื่อส่งข้อมูลเข้ามาได้)*
   * คลิกปุ่ม **Deploy** จากนั้นกดปุ่ม **ให้สิทธิ์เข้าถึง (Authorize Access)** -> เลือกบัญชี Gmail -> กด **ขั้นสูง (Advanced)** -> กด **ไปที่โครงการ (ไม่ปลอดภัย)** -> กด **อนุญาต (Allow)**
   * เมื่อทำเสร็จสิ้น คัดลอกค่า **URL ของเว็บแอป (Web App URL)** (ลิงก์จะลงท้ายด้วย `/exec`) เก็บเอาไว้

---

#### ทางเลือก B: ระบบฐานข้อมูล Supabase / PostgreSQL (แนะนำสำหรับทั้งโรงเรียน / ความเร็วสูง)
1. **สร้างโครงการใน Supabase:** สมัครใช้งานฟรีที่ [Supabase](https://supabase.com/) และสร้างโปรเจกต์ใหม่
2. **สร้างตารางข้อมูลใน SQL Editor:**
   * ไปที่เมนู **SQL Editor** ในแดชบอร์ด Supabase
   * สร้าง Query ใหม่ นำโค้ด SQL ทั้งหมดในไฟล์ [`supabase_schema.sql`](./supabase_schema.sql) ในโปรเจกต์นี้ ไปวางและกดปุ่ม **Run** เพื่อสร้างตารางทั้งหมด
3. **สร้างที่เก็บไฟล์รูปภาพหลักฐานใบลา (Storage Bucket):**
   * ไปที่เมนู **Storage** ใน Supabase
   * กดปุ่ม **Create Bucket** ตั้งชื่อว่า `evidence`
   * เลือกเปิดใช้งานเป็น **Public bucket** และกด Save
4. **คัดลอกข้อมูลเชื่อมต่อ:**
   * ไปที่ **Project Settings > API**
   * คัดลอกค่า **Project URL** และ **anon/public API Key** เก็บไว้

### ส่วนที่ 2: นำระบบเช็คชื่อขึ้นออนไลน์ผ่าน Vercel (1-Click Deploy)

คุณครูสามารถกดปุ่ม Vercel Deploy ด้านล่างเพื่อโคลนซอร์สโค้ดนี้และนำขึ้นออนไลน์ทันที:

[![Deploy with Vercel](https://vercel.com/button)](https://vercel.com/new/clone?repository-url=https%3A%2F%2Fgithub.com%2FJayjazzper%2FSmart-Attendance-System&env=NEXT_PUBLIC_SCHOOL_NAME,NEXT_PUBLIC_SCHOOL_DISTRICT,NEXT_PUBLIC_GOOGLE_SCRIPT_URL,NEXT_PUBLIC_SUPABASE_URL,NEXT_PUBLIC_SUPABASE_ANON_KEY)

**สิ่งที่ต้องทำในหน้าต่าง Vercel Deploy:**
1. **สมัครใช้งาน Vercel ฟรี** โดยล็อกอินผ่านบัญชี GitHub ของคุณ
2. ป้อนชื่อ Repository ส่วนตัวของคุณ Vercel จะช่วยโคลนโค้ดทั้งหมดนี้ไปสร้างเป็นโครงการของคุณใน GitHub โดยอัตโนมัติ
3. ป้อนค่าตัวแปรสภาพแวดล้อม (Environment Variables) ตามที่คุณครูเลือกใช้ในส่วนที่ 1:
   * **กรณีใช้ Google Sheets:**
     * `NEXT_PUBLIC_GOOGLE_SCRIPT_URL`: ลิงก์ URL ของเว็บแอปที่ได้จากส่วนที่ 1 (ทางเลือก A)
   * **กรณีใช้ Supabase (แนะนำ):**
     * `NEXT_PUBLIC_SUPABASE_URL`: ลิงก์ Project URL ที่คัดลอกมาจากส่วนที่ 1 (ทางเลือก B)
     * `NEXT_PUBLIC_SUPABASE_ANON_KEY`: รหัส anon/public API Key ที่คัดลอกมาจากส่วนที่ 1 (ทางเลือก B)
   * **ข้อมูลแสดงผลทั่วไป:**
     * `NEXT_PUBLIC_SCHOOL_NAME`: ชื่อโรงเรียนของคุณครู (เพื่อแสดงผลบนหัวเว็บ)
     * `NEXT_PUBLIC_SCHOOL_DISTRICT`: สังกัด/สำนักงานการศึกษาของโรงเรียน
4. คลิกปุ่ม **Deploy** และรอประมาณ 1 นาที ระบบจะแจ้งเตือนสำเร็จและให้ลิงก์ออนไลน์อย่างเป็นทางการพร้อมเข้าใช้งานบนคอมพิวเตอร์ แท็บเล็ต หรือมือถือได้ทันทีครับ!

### ส่วนที่ 3: ขั้นตอนการตั้งค่ากลุ่มไลน์แจ้งเตือน (LINE Notify)

เมื่อติดตั้งเว็บเสร็จแล้ว แอดมินต้องทำการผูกรหัสกลุ่มแจ้งเตือนไลน์ให้กับห้องเรียนแต่ละห้อง:
1. **เข้าสู่ระบบหลังบ้าน:** ไปที่หน้าเว็บโรงเรียนของคุณครู -> คลิกที่ **เครื่องมือระบบ (ฟันเฟือง)** -> เลือก **ตั้งค่าระบบ** (ล็อกอินด้วย Username: `admin` และ Password เริ่มต้นคือ `admin1234` หรือ `1234` ตามการตั้งค่าในไฟล์ประวัติของคุณครู)
2. **ออกโทเคนจาก LINE Notify:**
   * เปิดแท็บใหม่ เข้าไปที่เว็บ [LINE Notify My Page](https://notify-bot.line.me/my/) แล้วล็อกอินด้วยบัญชี LINE
   * กดปุ่ม **ออกโทเคน (Generate Token)**
   * ตั้งชื่อบอทแสดงตัว (เช่น *เช็คชื่อ ป.4/1*) จากนั้นค้นหากลุ่ม LINE ที่ต้องการส่งข้อความแจ้งเตือน แล้วคลิกเลือกกลุ่มนั้น และกด **Generate Token** จากนั้นคัดลอกรหัส Token ที่แสดงขึ้นมาเก็บไว้
3. **เชิญบอทเข้ากลุ่มแชท:** เปิดแอป LINE ในมือถือ แล้วเชิญบัญชีบอทที่ชื่อว่า **`LINE Notify`** เข้ามาในกลุ่มแชทไลน์ดังกล่าว
4. **บันทึกรหัสในระบบ:** นำรหัส Token ที่ได้มาวางลงในช่องห้องเรียนในหน้าตั้งค่าระบบของเว็บแอป และกดบันทึก คุณครูสามารถกดปุ่ม **ทดสอบส่งข้อความ** เพื่อทดสอบเชื่อมสัญญาณได้ทันที

## 🔧 การพัฒนาและรันบนเครื่องคอมพิวเตอร์ท้องถิ่น (Local Development)

หากต้องการรันเพื่อแก้ไขดีไซน์หรือพัฒนาโค้ดเพิ่มเติมในเครื่องคอมพิวเตอร์ของตัวเอง:

1. ติดตั้งไลบรารีทั้งหมดในโครงการ:
   ```bash
   npm install
   ```
2. ป้อนค่าตัวแปรสภาพแวดล้อมในไฟล์ `.env.local`:
   ```env
   NEXT_PUBLIC_GOOGLE_SCRIPT_URL=ลิงก์เว็บแอป_ของสคริปต์
   NEXT_PUBLIC_SUPABASE_URL=ลิงก์_Supabase_URL
   NEXT_PUBLIC_SUPABASE_ANON_KEY=รหัส_Supabase_Key
   ```
3. สั่งรันเวอร์ชันทดสอบบนเครื่อง:
   ```bash
   npm run dev
   ```
4. เปิดเบราว์เซอร์เข้าที่: [http://localhost:3000](http://localhost:3000)
